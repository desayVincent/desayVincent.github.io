如何使用Hexo + Next + github pages搭建个人博客

发表于 2024-10-07 更新于 2024-10-08 分类于 杂项

科普：Hexo , Next, Github Pages是什么

Hexo:

Hexo 是一个快速、简洁且高效的静态博客框架。主要特点：

  • 基于 Node.js 开发
  • 使用 Markdown 语法写作
  • 一键部署到各种平台
  • 丰富的插件系统
  • 强大的主题支持

主要功能：

  • 支持 Markdown 写作
  • 内置服务器，可本地预览
  • 支持多种部署方式（GitHub Pages, Heroku 等）
  • 强大的插件扩展系统
  • 支持多种主题切换

基本命令：
hexo new “文章标题” # 创建新文章
hexo server # 启动本地服务器
hexo generate # 生成静态文件
hexo deploy # 部署到远程站点

Next:

NexT 是 Hexo 最受欢迎的主题之一，以其简洁和功能丰富著称：

特点：

  • 设计简约精致
  • 响应式布局
  • 支持多种语言
  • 高度可定制
  • SEO 友好

主要功能：

  • 多种外观方案（Muse/Mist/Pisces/Gemini）
  • 支持多种第三方服务（评论、统计等）
  • 内置多种实用工具（搜索、标签云等）
  • 代码高亮
  • 数学公式支持

github pages

GitHub Pages 是 GitHub 提供的一个网页寄存服务，可以用于托管静态网站：

主要特点：

  • 免费托管静态网站
  • 自动从 GitHub 仓库部署
  • 支持自定义域名
  • 自动启用 HTTPS
  • 可靠的基础设施

使用方式：

  1. 个人/组织站点：username.github.io
  2. 项目站点：username.github.io/project

常见用途：

  • 个人博客
  • 项目文档
  • 作品展示
  • 静态网站

三者结合使用流程：

  1. 本地写作流程：

    • 使用 Hexo 创建博客框架
    • 安装并配置 NexT 主题
    • 用 Markdown 写作文章
    • 本地预览效果

  2. 部署流程：

    • Hexo 将 Markdown 转换为静态网页
    • 部署到 GitHub 仓库
    • GitHub Pages 自动发布网站

  3. 访问流程：

    • 读者通过 username.github.io 访问博客

    • GitHub Pages 提供网页服务

搭建个人博客

Step0：电脑环境

  • PC ubuntu1~20.04.2

  • npm 10.2.3

  • node v18.19.0

Step1：创建 github 账号和 .github.io 仓库

  1. 创建 GitHub 账号:

    如果您还没有 GitHub 账号，需要先在 GitHub 官网 (https://github.com/) 注册一个账号。

  2. 创建 .github.io 仓库:

    • 登录您的 GitHub 账号
    • 点击右上角的 “+” 图标，选择 “New repository”
    • 在 “Repository name” 框中，输入 “.github.io”
    • (将 替换为您的 GitHub 用户名，例如：如果您的用户名是 johndoe，则输入 johndoe.github.io)
    • 选择 “Public” 选项
    • 点击 “Create repository” 按钮

Step11：部署

  1. 生成个人访问令牌（Personal Access Token）：

    • 登录到 GitHub
    • 点击右上角的头像，然后选择 “Settings”
    • 在左侧菜单中，选择 “Developer settings”
    • 点击 “Personal access tokens”，然后选择 “Tokens (classic)”
    • 点击 “Generate new token”，然后选择 “Generate new token (classic)”
    • 给令牌一个描述性的名称
    • 选择令牌的有效期
    • 在 “Select scopes” 下，至少选中 “repo” 选项
    • 点击 “Generate token”
    • 复制生成的令牌（注意：这是您唯一能够看到令牌的机会，请务必保存）

  2. 使用个人访问令牌：

    • 在您的 Hexo 项目根目录中，编辑 _config.yml 文件

    • 找到 deploy 部分，修改如下：

    • 将 YOUR-USERNAME 替换为您的 GitHub 用户名

    • 将 YOUR-TOKEN 替换为您刚刚生成的个人访问令牌

  3. 重新部署：

    • 保存 _config.yml 文件

    • 运行

    • 这样应该就能解决认证问题，成功部署您的 Hexo 博客到 GitHub Pages

    • 记住，个人访问令牌就像密码一样重要，请妥善保管，不要分享给他人。如果您怀疑令牌被泄露，请立即在 GitHub 上撤销并生成新的令牌。
